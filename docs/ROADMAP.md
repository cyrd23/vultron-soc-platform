# Vultron Roadmap

## Phase 1 — Core Automation Platform (Completed)

The foundational Vultron backend has been established and includes:

- Elastic SIEM–backed telemetry ingestion
- Threat hunt pack framework
- Threat Hunter Agent
- Intel Enrichment Agent
- Triage Agent
- IR Investigation Agent
- Coordinator Agent
- Playbook Engine
- Detection Engineering Agent
- Master orchestrator
- Run ID–based artifact tracking
- Initial identity hunt coverage:
  - password_spray
  - oauth_consent_abuse
  - impossible_travel

---

## Phase 2 — Vultron Operations Portal (Highest Priority Next Step)

Build a web-based front end that turns Vultron into a usable analyst platform.

### Objectives

- Provide a central UI to operate the pipeline
- Allow analysts to review run history and artifacts
- Add approval workflows for response actions
- Enable AI-assisted question answering and investigation review

### Core Features

#### Pipeline Operations
- Run the full Vultron pipeline
- Run hunts by category
- Run individual hunt packs
- Re-run previous investigations

#### Investigation Workspace
- View run history and Run IDs
- Open a run and inspect:
  - hunt results
  - intel enrichment
  - triage decisions
  - IR findings
  - coordinator decisions
  - playbook execution plans

#### Entity Explorer
- Display and pivot on:
  - users
  - IPs
  - domains
  - hosts
  - applications

#### Detection Review
- View detection candidates generated by the platform
- Review Sigma rules
- Review Elastic ESQL detections
- Approve/export candidate detections

#### Playbook Approval
- Review recommended response steps
- Approve / reject / escalate actions
- Preserve audit trail of decisions

#### Dashboards
- Hunts executed
- Suspicious findings over time
- Top affected entities
- Detection candidates created
- Severity trends

#### AI Analyst Workspace
- Ask plain-English questions such as:
  - "Did any password spray IPs later authenticate successfully?"
  - "What were the most suspicious findings today?"
  - "Show me OAuth-related activity this week"

## Phase 3 — Expanded Hunt Coverage

Expand beyond identity hunts into additional threat domains.

### Identity
- MFA fatigue
- token abuse
- suspicious role assignments
- service principal abuse

### DNS / Command and Control
- rare domain access
- newly observed domains
- DNS tunneling
- DGA-like behavior

### Lateral Movement
- internal RDP
- remote service creation
- SMB movement
- abnormal admin behavior

### Endpoint
- suspicious PowerShell
- credential dumping
- persistence
- process injection

### Exposure
- vulnerable host activity
- internet-facing asset risk
- Tenable-driven exploitation validation

---

## Phase 4 — Attack Path Correlation

Introduce correlation logic to connect isolated hunts into attack chains.

### Examples
- password spray → successful login → mailbox activity
- OAuth consent → token abuse → SharePoint download
- vulnerable host → suspicious process → outbound beaconing

### Goals
- improve fidelity
- reduce false positives
- detect real attack progression
